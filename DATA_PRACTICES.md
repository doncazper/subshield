# Data Practices

Effective July 12, 2026

This document is the engineering counterpart to SubShield’s public privacy page. If the implementation and this document ever differ, the implementation must be corrected before deployment or this document must be updated before users are affected.

## Data inventory

| Data | Source | Purpose | Server retention | Browser retention | Shared externally |
|---|---|---|---|---|---|
| Temporary OAuth access token | Reddit OAuth | Authorize read-only API requests | Not stored; encrypted into the user’s cookie | Essential encrypted cookie, at most 1 hour | Reddit receives it on authorized API calls |
| OAuth state nonce | Generated by SubShield | Prevent login CSRF | Not stored | Essential encrypted cookie, at most 10 minutes | Returned to Reddit as OAuth state |
| Reddit username | `/api/v1/me` | Show connected account | None | Current page state | No |
| Moderated-community names | `/subreddits/mine/moderator` | Populate selector and validate authorization | None | Current page state | No |
| Up to 25 public submission titles, self-texts, domains, IDs, timestamps, permalinks | `/r/{subreddit}/new` | On-demand deterministic scan | None | Current scan state | No |
| Risk scores and reasons | Derived in request memory | Prioritize human review | None | Current scan state | No |
| Standard request/security metadata | Browser and Cloudflare | Deliver and protect the service | Controlled by Cloudflare’s hosting and security systems | Not applicable | Cloudflare as infrastructure provider |

## Explicit non-collection

SubShield does not collect or retain:

- Reddit passwords, emails, private messages, saved items, votes, browsing history, or private account settings
- refresh tokens or permanent Reddit authorization
- submission authors for the scoring or display workflow
- Reddit content in databases, KV, D1, R2, files, queues, logs, or analytics events
- cross-community histories or longitudinal user/subreddit profiles
- off-platform identifiers
- inferred health, political affiliation, sexual orientation, ethnicity, or other sensitive characteristics

## Processing boundaries

- A scan starts only after an authenticated moderator chooses a community and clicks the scan button.
- The approval-pending public demo uses only committed synthetic examples and makes no Reddit request.
- The Worker re-checks that the connected user moderates the requested community.
- Each scan is capped at 25 recent public submissions.
- Scan request bodies are measured in UTF-8 bytes and rejected above 2 KB; malformed JSON returns a bounded client error.
- Self-text is used only during scoring and is intentionally omitted from the response.
- Worker error logs contain only the request path and error class, never OAuth codes, tokens, usernames, community names, submissions, or scores.
- API responses set `Cache-Control: no-store`, `Pragma: no-cache`, and `X-Content-Type-Options: nosniff`.

## Deletion and revocation

- **Clear results** removes the current scan response from React state.
- **Log out** expires the encrypted session cookie immediately.
- The cookie expires automatically after the temporary Reddit token expires (at most one hour).
- Users can revoke the app from Reddit’s connected-app settings.
- There is no SubShield account or server-side Reddit-content record to delete.

## Change control

Any proposed change that introduces a new Reddit endpoint, scope, persistent store, background task, third-party transfer, AI/ML provider, or write action requires:

1. an update to this document and the public privacy page;
2. a code and threat-model review;
3. new tests;
4. Reddit approval before production use when required by Reddit policy.

See `THREAT_MODEL.md`, `docs/ARCHITECTURE.md`, and `REVIEWER_GUIDE.md` for the security boundaries, system map, and claim-to-evidence index.
